\begin{abstract}
Chain-of-thought supervision is usually written in natural language, even when
the latent reasoning problem is symbolic. We ask whether supervised language
models learn more reliable symbolic reasoning when the same derivation is
written instead as a compact formal trace. To isolate the trace substrate, we
construct paired synthetic tasks in which each instance has the same prompt,
answer, and latent proof, but two target traces: a formal derivation and a
natural language derivation. On branching proof chains, OLMo-7B trained on
depths up to 25 reaches 92.1\% \passsixteen{} correctness averaged over depths
30 to 50 and 83.3\% at depth 50 with formal traces, compared with 79.4\% and
51.0\% with natural language traces. On attribute constraints, formal traces
show a positive mean gap in answer correctness, but with high seed variance at
depth 50.
The direction is consistent across Qwen and Gemma model families, though some
architecture checks have high seed variance, and it becomes sharper when
training data contain shortcut distractors. Ablations show that the gain is not
recovered by longer traces, word-like predicates, pseudocode, explicit rule
annotations, or dual modality training. Invalid formal looking traces can still
preserve high answer accuracy while losing mechanical proof validity. Compact
formal traces are a better supervised substrate for learning correct symbolic
computations in these settings, but answer correctness and proof faithfulness
remain distinct objectives.
\end{abstract}

\section{Introduction}

Chain-of-thought methods improve language-model reasoning by asking the model to
produce intermediate steps before a final answer~\citep{wei2022chain,
kojima2022large}. Almost all such supervision is expressed in natural language.
That choice is useful for human readability, but it is not obviously optimal for
the model. Natural language is verbose, underspecified, and entangles the
underlying computation with discourse style. Symbolic tasks have another
natural representation: a compact formal derivation with explicit local rule
applications.

  \label{fig:overview}

The trace language can be varied without changing the problem. Holding the
latent proof fixed, should a model be supervised to reason in natural language
or in formal logic? We focus on supervised fine-tuning rather than prompting,
because the goal is to measure what models learn from the target trace itself.
In both conditions, the model generates the trace and the answer. Validators
are used only after generation, to measure whether a produced trace is
mechanically checkable.

We use controlled synthetic tasks because the causal comparison requires paired
supervision. Each instance is sampled from a latent derivation and rendered in
two ways. The first task, \branchproof{}, contains branching Horn-style proof
chains with coherent distractor branches and controlled proof depth. The second,
\attrcon{}, contains object-attribute constraint derivations. These tasks are
not meant to be broad reasoning benchmarks. They are instruments for testing
whether a trace substrate helps models learn compositional rule application,
state tracking, distractor rejection, and extrapolation to longer derivations.

Compact formal traces improve answer correctness most clearly when the proof is
longer than those seen in training. On \branchproof{}, the formal advantage is
strongest beyond the training depth range. On \attrcon{}, the same
answer-correctness pattern appears on a harder constraint-satisfaction surface,
although long-depth proof validity remains weak for both trace styles. The
effect persists across model families and under shortcut-rich training. We then
ask whether the result is explained by trace length, explicit rule labels,
code-like syntax, or the availability of both natural and formal modalities. The
controls do not support these simple explanations. Compact formal traces remain
the strongest representation among the variants we tested.

Correctness and faithfulness separate even in formal syntax. A generated trace
does not become a valid proof merely because it looks formal. Invalid
formal looking traces can keep high answer accuracy while formal validity
collapses. We therefore make a narrow claim. Formal traces improve supervised
learning of the correct symbolic computation in these controlled settings. They
do not by themselves guarantee that every generated trace is a faithful proof.

\textbf{Contributions and claims.} We isolate the trace substrate with paired
examples that hold prompts, answers, and latent derivations fixed. Compact
formal traces improve length extrapolation on \branchproof{} and show a
positive but noisier mean gap on \attrcon{}. The controls rule out several
simple explanations, including token budget, rule labels, code-like formatting,
shortcut markers, tested model family, and simply training both modalities. The
boundary is just as important: formal traces are a correctness substrate here,
not a guarantee of faithful proof generation.

\section{Related Work}

\subsection{Natural-language reasoning traces}

Chain-of-thought prompting showed that natural-language rationales can elicit
stronger reasoning from large models~\citep{wei2022chain}. Subsequent work
studies zero-shot reasoning prompts~\citep{kojima2022large}, rationale
bootstrapping~\citep{zelikman2022star}, decomposition into subproblems
~\citep{zhou2023least}, and search over alternative reasoning states
~\citep{yao2023tree}. These methods largely preserve natural language as the
reasoning medium. We instead vary the medium while holding the latent derivation
fixed.

Scratchpad supervision is an earlier form of the same idea: models can learn
multi-step computations more reliably when intermediate state is written into
the sequence~\citep{nye2021show}. Our question is not whether intermediate
state helps, but which representation of the same intermediate state is easiest
to learn.

\subsection{Faithfulness of reasoning traces}

Natural-language rationales need not be faithful descriptions of the computation
used by the model. Prior work shows that chain-of-thought explanations can
rationalize biased answers or omit features that influenced the prediction
~\citep{turpin2023language}, and proposes interventions for measuring CoT
faithfulness~\citep{lanham2023measuring}. Our trace-integrity controls make the
same distinction in a formal setting: a formal looking trace can support answer
accuracy without being a valid proof.

\subsection{Executable and symbolic substrates}

Program-aided language models and program-of-thought prompting use code as an
intermediate representation, often delegating computation to an interpreter
~\citep{gao2023pal,chen2023program}. Logic-based systems translate language
into formal representations and call symbolic solvers or verifiers
~\citep{pan2023logiclm,ling2023deductive}. These approaches demonstrate the
value of non-natural-language substrates, but they also change the execution
mechanism. Our experiments keep inference as plain next-token generation and
compare the supervised trace representation directly.

\subsection{Formal proof generation}

LLM-assisted theorem proving trains or prompts models to produce machine-checkable
proofs in systems such as Lean~\citep{polu2020generative,jiang2022draft,
leandojo2023,leanCopilot2024}. This literature is closest in spirit to our
formal-trace condition, but its target domain is mathematical theorem proving.
We use simpler generated tasks so that each latent derivation can be rendered as
either formal logic or natural language, enabling a controlled substrate
comparison.

\subsection{Shortcut learning}

Shortcut learning occurs when a model exploits correlations that work in the
training distribution but fail under shift~\citep{geirhos2020shortcut}. Our
generators let us create shortcut-rich training distributions while evaluating
on shortcut-neutral examples. This gives a direct test of whether a trace
language encourages rule following or brittle heuristic use.

\subsection{Synthetic compositional generalization and RLVR}

Synthetic compositional benchmarks such as SCAN and COGS are useful precisely
because they expose systematic generalization failures under controlled
distribution shifts~\citep{lake2018generalization,keysers2020measuring}. Our
tasks use the same philosophy, but the manipulated variable is the supervised
trace substrate. Recent RLVR critiques likewise emphasize that better pass@1
can reflect redistribution of existing samples rather than fundamentally new
reasoning capacity~\citep{yue2025does}. We therefore evaluate with sampled
\passsixteen{} and interpret gains as supervised sampling/correctness effects,
not as proof that the model has acquired a general theorem-proving procedure.

\section{Method}
\label{sec:method}

\subsection{Paired reasoning tasks}

Each example is generated from a latent proof. The prompt and final answer are
identical across trace conditions. Only the target reasoning trace changes.
\branchproof{} prompts contain facts and Horn-style rules over symbolic
predicates. Exactly one branch supports the queried conclusion. Other branches
are coherent distractors. \attrcon{} prompts contain objects, slots, values, and
constraints that propagate attributes through a derivation.

\subsection{Trace substrates}

The \logic{} trace uses compact symbolic atoms and explicit rule-application
lines. The \nl{} trace describes the same derivation in short controlled
sentences. Additional ablations modify this surface: symbol-padded logic,
wordified logic, pseudocode, rule-annotated natural language, shuffled traces,
invalid formal traces, same-datapoint hybrids, and mode-conditioned
dual modality training.

\subsection{Evaluation protocol}

We report \passsixteen{}, i.e. whether at least one of 16 sampled generations is
correct. The primary metric is final-answer correctness. We also report
correct-plus-valid joint metrics when validators support the trace surface. For
formal traces, validity is a mechanical proof check. For natural-language
traces, validity requires translation into the formal checker and is therefore a
diagnostic rather than a universal measure.

We use explicit depth terms throughout. The \emph{training range} contains
depths seen during fine-tuning. The
\emph{long-depth band} contains evaluated \branchproof{} depths greater than the
maximum training depth. For the main train-1-to-25 setting, this is depths
30, 35, 40, 45, and 50. The \emph{depth-50 endpoint} is the single hardest
evaluated proof depth. For \attrcon{}, we additionally report a
\emph{hard-depth band}, the fixed difficult slice used by that evaluator, and
the same depth-50 endpoint.

\section{Experimental Setup}
\label{sec:setup}

The main sweep fine-tunes OLMo-7B on \branchproof{} with maximum training depth
in $\{5,10,15,20,25\}$ over three seeds. The principal comparison is between
matched \logic{} and \nl{} targets. We repeat the train-1-to-25 comparison on
Qwen-2.5-1.5B, Qwen-2.5-7B, and Gemma-3-4B to check whether the result is
architecture-specific. Each train-depth subset contains 50k materialized
training examples. We fine-tune with LoRA rank 16, alpha 32, dropout 0.05, BF16
weights, batch size 1, learning rate $10^{-4}$, 10k optimizer steps, maximum
sequence length 8192, and adapters on attention and MLP projection modules.

For \attrcon{}, we use the same paired-rendering principle and report
three-seed train-1-to-25 results. The \attrcon{} numbers are used as a
replication of the answer-correctness effect on a different symbolic surface.
The depth-50 validity metrics are treated as diagnostics, not as evidence of
solved long-depth proof generation.

Post-hoc pass@k evaluation samples 16 generations per prompt at temperature
1.0 and top-$p=0.95$, stopping at \texttt{</answer>}. Unless otherwise noted,
each seed is evaluated on 32 examples per depth. \branchproof{} uses depths
$\{1,2,5,10,12,15,18,20,25,30,35,40,45,50\}$. The long-depth band for a
train-1-to-25 model is depths 30 to 50, i.e. 160 prompts per seed. \attrcon{} uses
the same depth grid and generation count, with a harder family-specific
hard-depth band. \branchproof{} generation caps are 4096 tokens for formal
traces and 6144 for natural-language traces. \attrcon{} uses 12288 for formal
traces and 8192 for natural-language traces, except one recovered formal row
with an 8192 cap. All tables report mean $\pm$ standard deviation across three
seeds unless a row explicitly states otherwise.

\passsixteen{} is computed per prompt: a prompt is correct if any of its 16
sampled generations yields the right final answer after extracting the
\texttt{<answer>} field. Depth-level metrics average prompts within a depth,
band metrics average the relevant depths within a seed, and table entries then
average across seeds. Formal joint validity requires answer correctness and a
mechanically valid formal trace. Natural-language joint validity requires answer
correctness and successful task-specific translation of the generated natural
trace into a valid formal proof.

\section{Analysis}
\label{sec:analysis}

\subsection{What the Ablations Rule Out}

The ablations are organized around specific objections. Is the effect just more
token exposure? Is it code-like formatting? Is it explicit rule annotation? Is
it simply that formal traces are longer or more structured? The controls do not
support those simple explanations, but they also do not isolate a single causal
feature. They narrow the explanation to a cluster of substrate properties:
compact symbols, local rule applications, low discourse load, and mechanical
grounding.

\section{Discussion}
\label{sec:discussion}

The trace substrate matters when the latent task is symbolic and the supervision
can be rendered from the same proof. Compact formal traces make the answer
computation easier to learn than natural-language traces in our controlled
settings. The effect is clearest under length extrapolation and shortcut shift,
where natural-language traces are more fragile.

The ablations narrow the plausible mechanism without identifying a single
causal feature. The effect is not recovered by more target-token exposure,
explicit rule labels, or a code-like wrapper. It is also not enough to train the
model on both substrates at once. The best representation we tested combines
compact symbolic atoms, local rule applications, and low discourse load. This
suggests that formal traces reduce the burden of learning through
natural-language surface variation and discourse dependencies.

The validity controls are equally important. A model can learn to answer
correctly from formal looking traces without producing a mechanically valid
proof. The current evidence therefore argues for formal traces as a better
\emph{training substrate for correctness}, not as a complete solution to
faithful reasoning. A natural next research direction is to combine compact
formal supervision with objectives or decoders that directly optimize trace
validity.

\section{Limitations}

The tasks are synthetic. This is necessary for a paired substrate comparison,
but it limits claims about open-domain reasoning. The main evidence comes from
supervised fine-tuning, not inference-time prompting or full-parameter
continued pretraining. The architecture checks are robustness tests rather than
a scaling law. Natural-language validity depends on task-specific translation
coverage, while formal validity does not imply causal faithfulness. Finally, the
current control set is extensive but not exhaustive. Other formal languages,
different proof granularities, and different decoding constraints may change the
tradeoffs.

\section{Conclusion}

The target language of reasoning supervision matters. In paired symbolic tasks,
compact formal traces improve extrapolative answer accuracy and resist shortcut
shift, while natural-language traces are more fragile. Formal syntax still does
not make a proof faithful, so correctness and validity must be trained or
decoded as separate objectives.

\section{Task Details}
\label{sec:task-details}

\branchproof{} samples a target proof path of controlled depth and adds
distractor branches that are locally coherent but do not support the queried
answer. Shortcut variants add training-only marker correlations that identify
the active branch with a chosen probability. Evaluation removes those markers.

\attrcon{} samples objects, slots, values, and propagation constraints. The
formal trace names slot/value relations directly. The natural trace describes
the same propagation in controlled prose. This changes the symbolic surface
while preserving the paired-rendering principle.

\section{Training and Evaluation Details}
\label{sec:training-details}

All main SFT runs use materialized datasets so that the formal and natural
conditions see the same prompts and answers. Each train-depth subset contains
50k examples, and the online validation subset contains 256 examples. Main runs
train for 10k optimizer steps with LoRA adapters on \texttt{q\_proj},
\texttt{k\_proj}, \texttt{v\_proj}, \texttt{o\_proj}, \texttt{up\_proj},
\texttt{down\_proj}, and \texttt{gate\_proj}. The LoRA rank is 16, alpha is 32,
dropout is 0.05, the optimizer learning rate is $10^{-4}$, the per-device batch
size is 1, gradient accumulation is 1 unless an ablation states otherwise, and
training uses BF16 with gradient checkpointing.

Post-hoc evaluation uses vLLM with top-$p=0.95$, temperature 1.0, 16 sampled
generations per prompt, and pass@k values $\{1,2,4,8,16\}$. The explicit depth
grid is $\{1,2,5,10,12,15,18,20,25,30,35,40,45,50\}$. For each seed and depth
we evaluate 32 prompts, so a train-1-to-25 \branchproof{} long-depth band
contains $5 \times 32=160$ prompts per seed. Reported uncertainty is seed-level
standard deviation, not a confidence interval.

\section{Trace Grammar and Validation}
\label{sec:grammar}

Formal traces contain four blocks: constants, predicates, premises, and proof
lines, followed by a conclusion and answer. A proof line contains a formula and
a rule justification such as premise retrieval or implication elimination. The
formal checker parses the generated blocks and verifies that each proof line is
licensed by earlier lines and premises. Natural-language traces use controlled
sentences for the same premises and proof steps. When possible, a
task-specific translator maps those sentences back into the formal checker.
Because translation coverage is task-specific, final-answer correctness is the
primary cross-task metric and translated validity is reported as a diagnostic.

The operational \branchproof{} formal surface is:

\begin{verbatim}
<formal>
<constants> CONST_DEF+ </constants>
<predicates> PRED_DEF+ </predicates>
<premises> FORMULA+ </premises>
<proof> PROOF_LINE+ </proof>
<conclusion> ATOM </conclusion>
<answer> LABEL </answer>

CONST_DEF  ::= CONST "=" LABEL
PRED_DEF   ::= PRED "x:" "x is" LABEL
ATOM       ::= PRED CONST
FORMULA    ::= ATOM | ATOM "&" ATOM "->" ATOM | ATOM "->" ATOM
PROOF_LINE ::= FORMULA "; R," REF
             | ATOM "; ->E," REF "," REF
             | ATOM "&" ATOM "; &I," REF "," REF
\end{verbatim}

The \attrcon{} surface uses the same block structure but replaces unary
predicate atoms with object-slot-value relations. In both tasks, references
must point to earlier proof lines or listed premises. Generated lines cannot
justify future steps.

\section{Generator and Evaluation Mechanics}
\label{sec:generator-eval}

The \branchproof{} generator samples a target path, assigns labels to constants
and predicates, then adds distractor branches that remain locally coherent but
do not derive the queried conclusion. In pseudocode:

\begin{verbatim}
sample target answer label
sample initial object state
for depth step in 1..D:
    sample active branch rule(s)
    derive next object state
    add coherent distractor rules for inactive branches
render one prompt
render the same latent proof as formal and natural-language targets
\end{verbatim}

The \attrcon{} generator follows the same paired-rendering principle with
objects, slots, values, and propagation constraints. The natural-language target
is not free-form prose. It is controlled text produced from the same latent
derivation.

Answer extraction first searches for the final \texttt{<answer>} field. If no
well-formed final answer is present, the generation is counted incorrect even
when earlier text contains the gold label. Formal validation then parses block
boundaries, declarations, formulas, proof-line references, and rule
applications. Common validation failures are malformed block structure,
undeclared symbols, references to unavailable lines, applying implication
elimination to the wrong antecedent, and stopping before the queried conclusion.
Natural-language validation adds one more possible failure mode: the controlled
sentence must translate back into a formal step before the proof checker can
evaluate it.

\section{Minimal Paired Example}
\label{sec:paired-example}

The following toy example shows the pairing used throughout the experiments.
The prompt and answer are identical. Only the target trace substrate changes.

\begin{verbatim}
Prompt:
a is ruby. a is north.
If a is north and a is ruby, then b is olive.
If b is olive, then b is east.
What is b?

Formal target:
<formal>
<constants> a = a b = b </constants>
<predicates> Rx: x is ruby Nx: x is north Ox: x is olive </predicates>
<premises> Ra Na Na & Ra -> Ob </premises>
<proof> Ra ; R,1
Na ; R,2
Na & Ra ; &I,5,4
Ob ; ->E,3,6 </proof>
<conclusion> Ob </conclusion>
<answer> olive </answer>

Natural-language target:
<think>
a is ruby. a is north.
Since a is north and a is ruby, b is olive.
Therefore, b is olive.
</think>
<answer> olive </answer>
\end{verbatim}

\section{Metric Definitions}

Answer correctness is computed after extracting the final answer field from the
generation. Formal joint validity requires both answer correctness and a
mechanically valid formal trace. Natural-language joint validity requires answer
correctness and successful translation of the generated natural-language trace
into a valid formal proof. Because the natural-language translator is
task-specific, answer correctness is the primary cross-task metric.
