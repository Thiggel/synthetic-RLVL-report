\begin{abstract}
Chain-of-thought supervision is usually written in natural language, even when
the latent reasoning problem is symbolic. We ask whether supervised language
models learn more reliable symbolic reasoning when the same derivation is
written instead as a compact formal trace. To isolate the trace substrate, we
construct paired synthetic tasks in which each instance has the same prompt,
answer, and latent proof, but two target traces: a formal derivation and a
natural-language derivation. On branching proof chains, OLMo-7B trained on
depths up to 25 reaches 92.1\% \passsixteen{} correctness averaged over depths
30--50 and 83.3\% at depth 50 with formal traces, compared with 79.4\% and
51.0\% with natural-language traces. On attribute constraints, formal traces
again improve answer correctness on the hard-depth band and at depth 50.
The effect is robust across Qwen and Gemma model families and becomes sharper
when training data contain shortcut distractors. Ablations show that the gain is
not recovered by longer traces, word-like predicates, pseudocode, explicit rule
annotations, or dual-modality training. At the same time, invalid formal-looking
traces can preserve high answer accuracy while losing mechanical proof validity.
The evidence therefore supports a precise conclusion: compact formal traces can
be a better supervised substrate for learning correct symbolic computations,
but answer correctness and proof faithfulness remain distinct objectives.
\end{abstract}

\section{Introduction}

Chain-of-thought methods improve language-model reasoning by asking the model to
produce intermediate steps before a final answer~\citep{wei2022chain,
kojima2022large}. Almost all such supervision is expressed in natural language.
That choice is useful for human readability, but it is not obviously optimal for
the model. Natural language is verbose, underspecified, and entangles the
underlying computation with discourse style. Symbolic tasks have another
natural representation: a compact formal derivation with explicit local rule
applications.

This paper studies the substrate question directly. Holding the latent proof
fixed, should a model be supervised to reason in natural language or in formal
logic? We focus on supervised fine-tuning rather than prompting, because the
goal is to measure what models learn from the target trace itself. In both
conditions, the model generates the trace and the answer. Validators are used
only after generation, to measure whether a produced trace is mechanically
checkable.

We use controlled synthetic tasks because the causal comparison requires paired
supervision. Each instance is sampled from a latent derivation and rendered in
two ways. The first task, \branchproof{}, contains branching Horn-style proof
chains with coherent distractor branches and controlled proof depth. The second,
\attrcon{}, contains object--attribute constraint derivations. These tasks are
not meant to be broad reasoning benchmarks. They are instruments for testing
whether a trace substrate helps models learn compositional rule application,
state tracking, distractor rejection, and extrapolation to longer derivations.

The main empirical result is that compact formal traces improve answer
correctness. On \branchproof{}, the formal advantage is strongest beyond the
training depth range. On \attrcon{}, the same answer-correctness pattern appears
on a harder constraint-satisfaction surface, although long-depth proof validity
remains weak for both trace styles. The effect persists across model families
and under shortcut-rich training. We then ask whether the result is explained by
trace length, explicit rule labels, code-like syntax, or the availability of
both natural and formal modalities. The answer is no: compact formal traces
remain the strongest representation among the variants we tested.

The paper also separates answer correctness from faithfulness. Formal syntax
does not make a generated trace a valid proof by itself. Invalid formal-looking
traces can keep high answer accuracy while formal validity collapses. We
therefore make a narrow claim. Formal traces improve supervised learning of the
correct symbolic computation in these controlled settings; they do not by
themselves guarantee that every generated trace is a faithful proof.

\section{Related Work}

\subsection{Natural-language reasoning traces}

Chain-of-thought prompting showed that natural-language rationales can elicit
stronger reasoning from large models~\citep{wei2022chain}. Subsequent work
studies zero-shot reasoning prompts~\citep{kojima2022large}, rationale
bootstrapping~\citep{zelikman2022star}, decomposition into subproblems
~\citep{zhou2023least}, and search over alternative reasoning states
~\citep{yao2023tree}. These methods largely preserve natural language as the
reasoning medium. We instead vary the medium while holding the latent derivation
fixed.

\subsection{Executable and symbolic substrates}

Program-aided language models and program-of-thought prompting use code as an
intermediate representation, often delegating computation to an interpreter
~\citep{gao2023pal,chen2023program}. Logic-based systems translate language
into formal representations and call symbolic solvers or verifiers
~\citep{pan2023logiclm,ling2023deductive}. These approaches demonstrate the
value of non-natural-language substrates, but they also change the execution
mechanism. Our experiments keep inference as plain next-token generation and
compare the supervised trace representation directly.

\subsection{Formal proof generation}

LLM-assisted theorem proving trains or prompts models to produce machine-checkable
proofs in systems such as Lean~\citep{polu2020generative,jiang2022draft,
leandojo2023,leanCopilot2024}. This literature is closest in spirit to our
formal-trace condition, but its target domain is mathematical theorem proving.
We use simpler generated tasks so that each latent derivation can be rendered as
either formal logic or natural language, enabling a controlled substrate
comparison.

\subsection{Shortcut learning}

Shortcut learning occurs when a model exploits correlations that work in the
training distribution but fail under shift~\citep{geirhos2020shortcut}. Our
generators let us create shortcut-rich training distributions while evaluating
on shortcut-neutral examples. This gives a direct test of whether a trace
language encourages rule following or brittle heuristic use.

\section{Method}
\label{sec:method}

\subsection{Paired reasoning tasks}

Each example is generated from a latent proof. The prompt and final answer are
identical across trace conditions; only the target reasoning trace changes.
\branchproof{} prompts contain facts and Horn-style rules over symbolic
predicates. Exactly one branch supports the queried conclusion; other branches
are coherent distractors. \attrcon{} prompts contain objects, slots, values, and
constraints that propagate attributes through a derivation.

\subsection{Trace substrates}

The \logic{} trace uses compact symbolic atoms and explicit rule-application
lines. The \nl{} trace describes the same derivation in short controlled
sentences. Additional ablations modify this surface: symbol-padded logic,
wordified logic, pseudocode, rule-annotated natural language, shuffled traces,
invalid formal traces, same-datapoint hybrids, and mode-conditioned
dual-modality training.

\subsection{Evaluation protocol}

We report \passsixteen{}, i.e. whether at least one of 16 sampled generations is
correct. The primary metric is final-answer correctness. We also report
correct-plus-valid joint metrics when validators support the trace surface. For
formal traces, validity is a mechanical proof check. For natural-language
traces, validity requires translation into the formal checker and is therefore a
diagnostic rather than a universal measure.

We use explicit depth terms throughout. The \emph{training range} contains
depths seen during fine-tuning. The
\emph{long-depth band} contains evaluated \branchproof{} depths greater than the
maximum training depth; for the main train-1-to-25 setting, this is depths
30, 35, 40, 45, and 50. The \emph{depth-50 endpoint} is the single hardest
evaluated proof depth. For \attrcon{}, we additionally report a
\emph{hard-depth band}, the fixed difficult slice used by that evaluator, and
the same depth-50 endpoint.

\section{Experimental Setup}
\label{sec:setup}

The main sweep fine-tunes OLMo-7B on \branchproof{} with maximum training depth
in $\{5,10,15,20,25\}$ over three seeds. The principal comparison is between
matched \logic{} and \nl{} targets. We repeat the train-1-to-25 comparison on
Qwen-2.5-1.5B, Qwen-2.5-7B, and Gemma-3-4B to check whether the result is
architecture-specific. All main tables report mean $\pm$ standard deviation over
three seeds.

For \attrcon{}, we use the same paired-rendering principle and report
three-seed train-1-to-25 results. The \attrcon{} numbers are used as a
replication of the answer-correctness effect on a different symbolic surface;
the depth-50 validity metrics are treated as diagnostics, not as evidence of
solved long-depth proof generation.

Table~\ref{tab:syntax} rules out several simple explanations. The formal model
does not win merely because it sees more useful target tokens: same-token formal
nearly preserves answer correctness but loses much of its validity. The gain is
also not recovered by making logic longer, replacing symbols with words, adding
rule labels to natural language, or wrapping steps in pseudocode. The most
consistent interpretation is that short symbolic atoms and local rule
applications make the supervised mapping easier to learn.

Hybrid traces answer a natural objection: formal and natural language may be
complementary. The results do not support that hypothesis
(Table~\ref{tab:hybrid}). Formal-first hybrids are much better than

fall short of formal-only training on depth-50 correctness. Separate
mode-conditioned examples remove the same-target length issue and train longer,
but still do not recover the formal-only baseline.

\section{Discussion}
\label{sec:discussion}

The experiments support a specific claim about supervised reasoning. When the
latent task is symbolic and the supervision can be rendered from the same proof,
the trace substrate matters. Compact formal traces make the answer computation
easier to learn than natural-language traces in our controlled settings. The
effect is clearest under length extrapolation and shortcut shift, where
natural-language traces are more fragile.

The ablations narrow the mechanism. The useful ingredient is not simply more
tokens, more explicit rule labels, or a code-like wrapper. It is also not enough
to train the model on both substrates at once. The best representation we tested
uses compact symbolic atoms and local formal rule applications. This suggests
that formal traces reduce the burden of learning through natural-language
surface variation and discourse dependencies.

The validity controls are equally important. A model can learn to answer
correctly from formal-looking traces without producing a mechanically valid
proof. The current evidence therefore argues for formal traces as a better
\emph{training substrate for correctness}, not as a complete solution to
faithful reasoning. A natural next research direction is to combine compact
formal supervision with objectives or decoders that directly optimize trace
validity.

\section{Limitations}

The tasks are synthetic. This is necessary for a paired substrate comparison,
but it limits claims about open-domain reasoning. The main evidence comes from
supervised fine-tuning, not inference-time prompting or full-parameter
continued pretraining. The architecture checks are robustness tests rather than
a scaling law. Natural-language validity depends on task-specific translation
coverage, while formal validity does not imply causal faithfulness. Finally, the
current control set is extensive but not exhaustive; other formal languages,
different proof granularities, and different decoding constraints may change the
tradeoffs.

\section{Conclusion}

Natural language is not the only possible substrate for chain-of-thought
supervision. In controlled paired symbolic tasks, compact formal traces improve
answer correctness, especially beyond the training depth range and under
shortcut-rich training. The result repeats across task surfaces and model
families, and it is not reproduced by length, pseudocode, rule labels, or
dual-modality training. Formal traces help models learn the right computation,
but validity and faithfulness remain separate problems.

\section{Task Details}

\branchproof{} samples a target proof path of controlled depth and adds
distractor branches that are locally coherent but do not support the queried
answer. Shortcut variants add training-only marker correlations that identify
the active branch with a chosen probability. Evaluation removes those markers.

\attrcon{} samples objects, slots, values, and propagation constraints. The
formal trace names slot/value relations directly; the natural trace describes
the same propagation in controlled prose. This changes the symbolic surface
while preserving the paired-rendering principle.

\section{Metric Definitions}

Answer correctness is computed after extracting the final answer field from the
generation. Formal joint validity requires both answer correctness and a
mechanically valid formal trace. Natural-language joint validity requires answer
correctness and successful translation of the generated natural-language trace
into a valid formal proof. Because the natural-language translator is
task-specific, answer correctness is the primary cross-task metric.
