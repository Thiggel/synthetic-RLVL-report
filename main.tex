\begin{abstract}
Chain-of-thought supervision is usually expressed in natural language, even for
tasks whose latent structure is symbolic. We study whether a language model
learns more reliable reasoning when the same latent proof is supervised in a
compact formal language instead. To isolate the representation variable, we
generate paired synthetic tasks in which each instance has the same prompt,
answer, and latent derivation, but two target traces: one formal and one
natural-language. Across branching proof chains and attribute constraints,
formal traces improve answer correctness under length extrapolation. With
OLMo-7B trained on proof chains up to depth 25, formal traces reach
92.1\% held-out correctness and 83.3\% depth-50 correctness, compared with
79.4\% and 51.0\% for natural-language traces. The effect is robust across
Qwen and Gemma models and is strongest when training data contain shortcut
distractors. Ablations show that the gain is not recovered by longer traces,
wordified predicates, pseudocode, rule-annotated natural language, or
dual-modality training. At the same time, invalid formal-looking traces can
preserve high answer accuracy while losing mechanical proof validity. The
evidence therefore supports a precise claim: compact formal traces can be a
better supervised substrate for learning correct symbolic reasoning, while
validity and faithfulness remain separate objectives.
\end{abstract}

\section{Introduction}

Chain-of-thought methods ask language models to expose intermediate reasoning
before committing to an answer~\citep{wei2022chain,kojima2022large}. The
reasoning trace is almost always written in natural language. This is a sensible
default for human readability, but it also introduces surface variation,
ambiguity, and long-range discourse dependencies. For symbolic reasoning tasks,
there is a natural alternative: write the same derivation in a compact formal
language.

This paper asks a narrow empirical question. Holding the underlying proof
fixed, which target language produces better supervised reasoning behavior:
natural-language chain of thought, or formal-logic chain of thought? We study
this under supervised fine-tuning, not prompting, because our goal is to measure
what small and mid-sized models can learn from the trace representation itself.
The comparison is not between a neural model and an external solver. The model
generates the trace and answer in both conditions; validators are used only for
measurement.

We use controlled synthetic tasks rather than natural benchmarks because the
causal comparison requires paired supervision. Each instance is generated from
a latent proof and rendered twice: once as a formal derivation and once as a
natural-language derivation. The main task family consists of branching proof
chains: Horn-style rules, coherent distractor branches, and a target conclusion
whose depth is controlled by the generator. The second family consists of
attribute constraints over objects, slots, and values. These tasks are not
intended to be broad benchmarks. They are instruments for testing whether a
reasoning substrate helps models learn compositional rule application, state
tracking, distractor rejection, and extrapolation to longer derivations.

The main result is that formal traces improve answer correctness. On branching
proof chains, formal supervision is substantially stronger at depth 50, far
beyond the training range. On attribute constraints, the held-out gap is smaller
but the depth-50 gap remains large. The rest of the paper tests whether this is
an artifact of trace length, explicit rule labels, syntax, shortcuts, hybrid
targets, or architecture choice. The answer is mostly no: compact formal syntax
is the strongest representation among the variants we tried.

The result also has an important limitation. Formal-looking traces are not
automatically faithful proofs. Negative controls with invalid or shuffled
formal traces can keep high answer accuracy while mechanical validity collapses.
We therefore separate the correctness claim from the stronger claim that the
model's trace is a causally faithful proof. Our experiments establish the
former more clearly than the latter.

\section{Related Work}

\subsection{Natural-language reasoning traces}

Chain-of-thought prompting showed that intermediate rationales can substantially
improve large-model performance on arithmetic, commonsense, and symbolic
reasoning tasks~\citep{wei2022chain}. Subsequent work studies zero-shot
reasoning cues~\citep{kojima2022large}, iterative rationale
bootstrapping~\citep{zelikman2022star}, decomposition into subproblems
~\citep{zhou2023least}, and search over alternative reasoning states
~\citep{yao2023tree}. These methods largely keep natural language as the
reasoning medium. We instead vary the medium while holding the latent proof
fixed.

\subsection{Executable and symbolic substrates}

Program-aided language models and program-of-thought prompting use code as an
intermediate representation, often delegating computation to an interpreter
~\citep{gao2023pal,chen2023program}. Logic-based systems translate language
into formal representations and call symbolic solvers or verifiers
~\citep{pan2023logiclm,ling2023deductive}. These approaches demonstrate the
value of non-natural-language substrates, but they also change the execution
mechanism. Our experiments keep generation-only inference and compare the
learned trace representation directly.

\subsection{Formal proof generation}

LLM-assisted theorem proving trains or prompts models to produce machine-checkable
proofs in systems such as Lean~\citep{polu2020generative,jiang2022draft,
leandojo2023,leanCopilot2024}. This literature is closest in spirit to our
formal-trace condition, but its target domain is mathematical theorem proving.
We use much simpler generated tasks so that the same derivation can be rendered
as either formal logic or natural language, enabling a controlled substrate
comparison.

\subsection{Shortcut learning}

Shortcut learning occurs when models rely on correlations that work in the
training distribution but fail under shift~\citep{geirhos2020shortcut}. Our
generators let us create shortcut-rich training distributions while evaluating
on shortcut-neutral data. This gives a direct test of whether a trace language
encourages the intended rule-following computation or a brittle heuristic.

\section{Experimental Setup}
\label{sec:setup}

\subsection{Tasks}

The primary task family is \emph{branching proof chains}. A prompt contains
facts and Horn-style rules over symbolic predicates. Exactly one chain supports
the queried conclusion; other branches are locally coherent distractors. The
generator controls proof depth, branch structure, and shortcut markers. Formal
targets consist of compact derivation lines with rule applications. Natural
targets describe the same derivation in short sentences. Both targets end with
the same answer tag.

The secondary family is \emph{attribute constraints}. Instances contain objects
with slot/value attributes and rules that propagate values through a derivation.
This family tests whether the formal-trace effect survives a more semantic
surface where symbols are closer to words and correctness depends on preserving
object--attribute bindings.

More generator detail and examples are given in Appendix~\ref{app:tasks}.

\subsection{Models and evaluation}

The main sweep fine-tunes OLMo-7B on branching proof chains with maximum
training depth in $\{5,10,15,20,25\}$ over three seeds. Evaluation reports
\passk{} answer correctness on held-out examples and on a sparse depth-50 split.
We also report joint correctness-plus-validity where the validator supports the
trace surface. For formal traces this is mechanical formal validity. For
natural-language traces this is translated validity and is therefore only
meaningful for task families with a supported translator.

All tables report mean $\pm$ standard deviation over seeds unless otherwise
specified. We use plots only for depth or shortcut-rate trends; ablations are
reported as tables to make the comparisons explicit.

Table~\ref{tab:syntax} rules out several simple explanations. Formal traces do
not win because they are longer; token-budget matching and symbol padding do
not recover the result. They do not win because they look more like code;
pseudocode remains far below compact formal traces. They do not win merely
because rules are visible; rule-annotated natural language is also weaker. The
most consistent explanation is that compact symbolic atoms and local rule
applications make the supervised mapping easier to learn.

Same-datapoint hybrids answer a natural objection: perhaps formal and natural
language are complementary. Table~\ref{tab:hybrid} suggests otherwise. Putting
the formal trace first is much better than putting natural language first, but
formal-only training remains the strongest correctness condition. Separate
mode-conditioned examples remove the same-target length issue, but they also do
not recover the formal-only baseline. The likely failure modes are
representation interference and reduced per-substrate exposure rather than a
simple train-test prompt mismatch.

\section{Discussion}

The results support a modest but useful conclusion. For these controlled
symbolic tasks, compact formal traces are a better supervised representation
for learning answer correctness than natural-language traces. The evidence is
strongest under depth extrapolation and shortcut shift, where natural-language
traces are more fragile. The ablations narrow the explanation: the gain is not
caused by trace length, explicit rule labels, pseudocode, or mixing formal and
natural language.

The paper should not be read as claiming that formal chain of thought is always
faithful. The invalid-trace controls show that answer correctness can survive
even when proof validity is destroyed. This is a central distinction. Formal
syntax appears to help the model learn the correct computation, but validity
and causal faithfulness require additional objectives or verification.

\section{Limitations}

The tasks are synthetic. This is necessary for the paired substrate comparison,
but it limits claims about open-domain reasoning. The main model is OLMo-7B,
with architecture checks rather than a full scaling law. Natural-language
translated validity remains family-specific, and the attribute-constraint
translator does not remove the long-depth failure mode: depth-50 translated
joint validity remains weak. The 32B result is only a short-context sanity
check. Finally, the current terse-natural control is not a clean compression
control because the default natural-language traces are already terse.

\section{Conclusion}

When the latent problem is symbolic, the language used for chain-of-thought
supervision matters. In our controlled paired setting, compact formal traces
yield higher answer correctness than natural-language traces, especially under
depth extrapolation and shortcut-rich training. The effect survives architecture
checks and is not reproduced by superficial variants of the trace. The next
step is to connect this correctness benefit to stronger guarantees about proof
validity and faithfulness.

\section{Task Generation Details}
\label{app:tasks}

Branching proof chains generate a finite set of symbols, facts, and Horn-style
rules. A target path of depth $d$ is sampled, and distractor branches are added
so that local facts and rules remain plausible but do not derive the queried
answer. The training split samples depths up to a configured maximum; the
extrapolation split evaluates depth 50. Shortcut variants add marker features
that are predictive during training and removed at evaluation.

Attribute constraints use object identities, slots, and values. A derivation
propagates a value through a sequence of constraints. Formal traces name the
slot/value relations directly; natural-language traces describe the same
propagation in words. This task changes the ontology while preserving the
paired rendering principle.

Answer correctness is computed after extracting the final answer field from the
generation. Formal validity checks whether the generated proof lines can be
validated against the task facts and rules. Natural-language joint validity is
reported only when the controlled translator supports the task family. We
therefore treat correctness as the primary cross-family metric and use validity
to diagnose whether a high-correctness trace is also mechanically checkable.
